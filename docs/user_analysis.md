# User Analysis — Campus Event Management System

## 1. Target Users

### User Group 1: Students (Primary Users)
Students are the main audience. They want to discover what's happening on campus, sign up for activities, and share their experiences.

**Tasks:**
1. Browse and search upcoming campus events
2. Filter events by category (workshop, seminar, sports, club)
3. Register for an event by filling in personal details
4. View event details (date, time, venue, available spots)
5. Submit feedback and ratings after attending an event

### User Group 2: Event Organisers / Staff (Secondary Users)
Staff members create events and need to know how students feel about them so they can improve future events.

**Tasks:**
1. Post new events with descriptions and logistics
2. View registrations and attendee count
3. Read student feedback and ratings
4. Identify popular events and trends
5. Update or cancel events

---

## 2. Three Main User Goals

| # | User Goal | Why It Matters (HCI Relevance) |
|---|-----------|-------------------------------|
| 1 | **Quickly find relevant events** | Users should not be overwhelmed. Filters, search, and clear categories reduce cognitive load (Simplicity). |
| 2 | **Register for events without errors** | A smooth registration with inline validation prevents frustration and data mistakes (Error Prevention). |
| 3 | **Leave feedback easily after attending** | A simple star rating plus text box encourages participation and gives instant visual feedback (Visibility & Feedback). |

---

## 3. Why These Goals Matter for HCI

These goals directly map to the four HCI principles required by the assignment:

- **Goal 1 → Simplicity**: A clean event listing with filters and search lets users find what they want without unnecessary complexity.
- **Goal 2 → Error Prevention**: Real-time form validation, format hints, and a disabled submit button until all fields are valid prevent users from making mistakes.
- **Goal 3 → Visibility & Feedback**: Star ratings give immediate visual response, and toast/modal confirmations tell users their action succeeded.
- **All Goals → Consistency & Standards**: The same button styles, colour coding, and navigation pattern are used across every screen, so users always know where they are.
