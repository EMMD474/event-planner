Screen 2 of 3
Event Registration Modal
Opens as an overlay on the events page when "Register" is clicked. The event is pre-selected from the card.
✕
Register Now → (disabled until all fields valid)
A — Pre-selection: The event dropdown is pre-filled with the event from the card that was clicked. Users can change it.
B — Real-time validation: Green border = valid. Red border + error message below = invalid. Student ID shows a format hint ("STU followed by 7 digits") at all times.
C — Disabled submit: "Register Now" button is greyed out and non-clickable until every field passes validation — preventing incomplete submissions.
D — Overlay close: Clicking ✕ or the backdrop closes the modal and returns to the event listing — no navigation needed.
Screen 3 of 3
Event Feedback Modal
Opens as an overlay when "Give Feedback" is clicked. Users select an event, rate it, and write a comment.
✕
Submit Feedback →
A — Event pre-selection: Same as registration — the event from the clicked card is pre-filled in the dropdown.
B — Star rating: Stars highlight gold on hover. Clicking locks the selection and updates the label (e.g., "Very Good — 4/5"). "Click a star to rate" prompt is visible until a star is selected.
C — Character counter: Live counter shows characters used out of 500. Input is capped at 500; minimum is 10 characters for the form to become valid.
D — Submit enabled: Button activates only when event, rating, and comment are all valid. On submit, the modal closes and a toast notification confirms the action.
